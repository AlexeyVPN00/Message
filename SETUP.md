## 🔧 Первоначальная настройка

### 1. Запуск Docker контейнеров

```bash
# Находясь в корневой папке проекта
docker-compose up -d
```

Проверьте, что контейнеры запущены:
```bash
docker ps
```

Должны быть запущены:
- `messenger_postgres` (порт 5433)
- `messenger_redis` (порт 6380)

### 2. Установка зависимостей

**Backend:**
```bash
cd backend
npm install
```

**Frontend:**
```bash
cd frontend
npm install
```

### 3. Настройка переменных окружения

**Backend** - файл `backend/.env` уже настроен:
```env
PORT=5000
NODE_ENV=development

DB_HOST=localhost
DB_PORT=5433
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_DATABASE=messenger

REDIS_HOST=localhost
REDIS_PORT=6380

JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d

FRONTEND_URL=http://localhost:3000
```

**Frontend** - создайте файл `frontend/.env`:
```env
VITE_API_URL=http://localhost:5000/api
VITE_WS_URL=http://localhost:5000
```

### 4. Создание тестовых пользователей

```bash
cd backend
npm run seed
```

Это создаст 4 тестовых пользователя:
- **Email:** admin@test.com | **Пароль:** admin123
- **Email:** user1@test.com | **Пароль:** user123
- **Email:** user2@test.com | **Пароль:** user123
- **Email:** test@test.com | **Пароль:** test123

## 📱 Основные возможности для тестирования

### 1. Регистрация нового пользователя
- Перейдите на страницу регистрации
- Заполните форму
- Система проверит уникальность email, username и телефона

### 2. Создание чата
- Войдите как `user1@test.com`
- Перейдите в раздел "Диалоги"
- В поле поиска введите `@user2` или `@admin`
- Кликните на найденного пользователя
- Начните общение

### 3. Добавление в контакты
- Найдите пользователя через поиск
- Откройте его профиль (клик на имя пользователя)
- Нажмите "Добавить в контакты"
- Перейдите в раздел "Контакты" и увидите добавленного пользователя

### 4. Групповые чаты
- В разделе "Диалоги" нажмите "+" (создать групповой чат)
- Введите название группы
- Выберите участников
- Создайте чат

### 5. Смена темы
- В боковом меню внизу есть кнопка переключения темы (🌙/☀️)
- Переключайтесь между светлой и темной темой

## 🐛 Возможные проблемы и решения

### Backend не запускается

**Проблема:** `Error: listen EADDRINUSE: address already in use :::5000`
**Решение:** Порт 5000 уже занят. Остановите другое приложение или измените порт в `.env`

**Проблема:** `Database connection failed`
**Решение:**
1. Проверьте, что Docker контейнеры запущены: `docker ps`
2. Перезапустите контейнеры: `docker-compose restart`

### Frontend не подключается к Backend

**Проблема:** Ошибки CORS или Network Error
**Решение:**
1. Убедитесь, что backend запущен на порту 5000
2. Проверьте файл `frontend/.env` - должен быть `VITE_API_URL=http://localhost:5000/api`
3. Очистите кэш браузера и перезагрузите страницу

### WebSocket не работает

**Проблема:** Сообщения не приходят в реальном времени
**Решение:**
1. Проверьте консоль браузера (F12) на ошибки WebSocket
2. Убедитесь, что `VITE_WS_URL=http://localhost:5000` в frontend/.env
3. Перезапустите backend сервер

### Не могу войти в систему

**Проблема:** "Invalid email or password"
**Решение:**
1. Убедитесь, что запустили seed: `npm run seed` в папке backend
2. Проверьте правильность email и пароля
3. Попробуйте зарегистрировать новый аккаунт

## 📊 Структура базы данных

Все таблицы созданы автоматически через TypeORM:
- `users` - пользователи
- `chats` - чаты (личные и групповые)
- `chat_members` - участники чатов
- `messages` - сообщения
- `contacts` - контакты пользователей
- `channels` - каналы
- `posts` - посты в ленте
- `notifications` - уведомления
- `file_attachments` - загруженные файлы
